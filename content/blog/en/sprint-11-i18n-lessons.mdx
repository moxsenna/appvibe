## Context

Our agency site defaults to **Indonesian**, with **English** under `/en/...`. Portfolio and demo **slugs stay canonical** — less maintenance and fewer broken links.

## Data pattern

```ts
export type Localized<T> = { id: T; en: T };

// Consumer
const { lang } = useLang();
<h2>{service.title[lang]}</h2>
```

UI chrome (navbar, section eyebrows) lives in `dict.pages.*`. Long-form content (services, portfolio, demos) stays in `src/data/*` with `Localized` fields.

## SEO

Each page calls `applyPageMeta` with **per-language title/description** and `paths: { id, en }` for **hreflang**.

## Lessons

1. Change data and consumers **in the same batch** — otherwise React error #31 (`[object Object]`).
2. Run `typecheck` at every major milestone.
3. Demo niches span dozens of files — Tier B i18n needs discipline, not one heroic sweep.

If you are building a bilingual marketing site, start with **routes + one page**, then data.
