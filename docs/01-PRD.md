# 01-PRD.md — AppVibe Studio Website & Portfolio System

## 1. Ringkasan Produk

**Nama produk:** AppVibe Studio Website  
**Jenis:** Website utama agency + portfolio/demo interaktif  
**Target utama:** Pengusaha, UMKM, bisnis jasa, klinik, edukasi, properti, dan bisnis lokal yang belum punya website proper.

AppVibe Studio adalah website utama untuk memasarkan jasa pembuatan website, landing page, web app ringan, dashboard bisnis, dan integrasi automation berbasis workflow modern/vibecoding.

Website ini harus menjadi **sales asset**, bukan sekadar company profile. Fungsi utamanya adalah membuat calon klien memahami:

1. AppVibe bisa membuat website yang terlihat profesional.
2. AppVibe bisa membuat demo/app yang terasa siap pakai.
3. AppVibe memahami kebutuhan bisnis, bukan hanya desain.
4. Calon klien bisa langsung konsultasi via WhatsApp.

---

## 2. Latar Belakang

Banyak pengusaha kecil dan menengah belum memiliki website proper. Mereka masih mengandalkan WhatsApp, Instagram, marketplace, brosur, atau link sederhana. Akibatnya:

- bisnis terlihat kurang profesional,
- calon customer sulit memahami layanan,
- proses inquiry masih manual,
- data leads tidak tertata,
- promosi sulit diukur,
- kepercayaan calon pelanggan belum maksimal.

Di sisi lain, banyak jasa website hanya menampilkan screenshot statis atau template standar. AppVibe Studio harus tampil beda dengan menunjukkan demo portfolio yang:

- responsif,
- punya mock data realistis,
- punya user flow,
- punya copy bisnis,
- terlihat seperti website/app sungguhan,
- mudah dipahami pemilik bisnis non-teknis.

---

## 3. Tujuan Produk

### 3.1 Business Goals

1. Membangun trust kepada calon klien.
2. Menjelaskan layanan AppVibe secara sederhana dan meyakinkan.
3. Menghasilkan inquiry/konsultasi via WhatsApp.
4. Menunjukkan kemampuan membuat website, landing page, dashboard, dan mini app.
5. Menjadi portfolio utama untuk kegiatan sales.
6. Menjangkau pengusaha yang belum punya website proper.
7. Membantu calon klien membayangkan hasil akhir sebelum membeli jasa.

### 3.2 Product Goals

1. Homepage visual kuat dan copy menjual.
2. Portfolio/demo tidak setengah jadi.
3. Setiap demo responsif dan punya mock data realistis.
4. Setiap portfolio punya case study.
5. Semua CTA jelas mengarah ke WhatsApp/kontak.
6. Struktur konten mudah dikembangkan.
7. Website mudah dikerjakan oleh coding agent/vibecoding.

---

## 4. Non-Goals

Untuk MVP, produk ini **bukan**:

1. Marketplace jasa.
2. Client portal.
3. Sistem invoice/pembayaran jasa.
4. SaaS multi-tenant.
5. CMS kompleks.
6. Dashboard internal real-time.
7. Sistem project management.
8. Sistem login untuk calon klien.
9. Blog besar dengan CMS.
10. Full backend application.

MVP fokus pada **website agency + portfolio/demo yang menjual**.

## 14. Acceptance Criteria

### Homepage

- Pengunjung memahami layanan dalam 5 detik.
- Ada headline, subheadline, CTA, visual mockup.
- Minimal 4 layanan tampil.
- Minimal 5 portfolio tampil.
- CTA WhatsApp bekerja.
- Mobile responsive.

### Portfolio

- 5 portfolio awal tampil.
- Filter kategori bekerja.
- Setiap portfolio punya detail page.
- Setiap portfolio punya demo page.

### Demo

- Tidak ada lorem ipsum.
- Mock data realistis.
- Responsive.
- CTA konsultasi tersedia.

### Case Study

- Menjelaskan problem, solution, features, flow, business value.
- Mudah dipahami non-teknis.

### Contact

- CTA WhatsApp berjalan.
- Form terlihat jelas.
- Required fields jelas.
