# 05-PORTFOLIO-DEMO-SPEC.md — Portfolio and Demo Specification

## 1. Purpose

Dokumen ini mendefinisikan portfolio dan demo yang harus dibuat untuk AppVibe Studio MVP.

Portfolio/demo adalah aset penjualan utama. Demo tidak boleh setengah jadi. Setiap demo harus terlihat seperti contoh produk sungguhan yang bisa langsung dipahami oleh calon klien.

Targetnya:

> Calon klien melihat demo dan berpikir: “Bisnis saya juga butuh seperti ini.”

---

## 2. Portfolio System Overview

MVP memiliki 5 portfolio utama:

1. Company Profile Jasa Profesional
2. Landing Page Webinar / Campaign
3. Website Klinik / Health / Beauty
4. Website Properti / Konstruksi
5. Dashboard Lead Management

Setiap portfolio harus punya:

- card di halaman portfolio,
- case study page,
- live demo page,
- mock data realistis,
- CTA konsultasi,
- visual thumbnail,
- business value.

---

## 3. Shared Portfolio Requirements

### 3.1 Portfolio Card

Each card includes:

- thumbnail,
- category badge,
- title,
- niche,
- short summary,
- feature tags,
- business outcome,
- CTA: Lihat Studi Kasus,
- CTA: Buka Demo.

### 3.2 Case Study Page Structure

Each case study must include:

1. Hero
2. Business context
3. Business problem
4. Solution
5. Key features
6. User flow
7. Screens preview
8. Mock data highlights
9. Business value
10. Tech capability
11. CTA

### 3.3 Demo Page Shared Requirements

Every demo page must include:

- demo business name,
- demo navbar,
- hero,
- niche-relevant content,
- services/products/listing/data,
- testimonials,
- FAQ,
- contact/CTA,
- AppVibe demo CTA,
- responsive layout.

### 3.4 Strict Rules

1. No lorem ipsum.
2. No half-finished section.
3. No broken CTA.
4. No fake “coming soon” inside MVP demo.
5. No blank data cards.
6. No raw technical jargon.
7. Demo must look good on mobile.

---
