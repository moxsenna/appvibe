## 2. Business Context

### Masalah bisnis sebelum punya website
- Calon klien hanya melihat Instagram atau menerima PDF — tidak ada **satu link resmi**.
- Tim **menjelaskan layanan berulang** di WhatsApp.
- Portfolio proyek tersebar — **sulit ditunjukkan saat meeting pertama**.
- Bisnis berpengalaman tapi **terlihat kurang kredibel** online.
- Inquiry masuk tanpa konteks kebutuhan, budget, atau jenis layanan.

### Tujuan website
Wajah digital resmi Arunika Konsultan: positioning, layanan, bukti kerja, proses, dan cara menghubungi — nyaman di HP.

### Conversion goals
| Prioritas | Goal |
|-----------|------|
| Utama | Inquiry berkualitas via form atau WhatsApp dengan konteks |
| Sekunder | Visitor paham layanan sebelum chat; bookmark link untuk tim internal |

### Peran website dalam sales process
| Tahap | Peran |
|-------|-------|
| Awareness | Link dari bio, kartu nama, iklan, referral |
| Consideration | Layanan + portfolio + proses kerja |
| Trust-building | Studi kasus + FAQ + testimoni skenario |
| Action | Form inquiry / WhatsApp |
| Follow-up | Tim punya data awal sebelum meeting |

### Keraguan calon klien & jawaban website
| Keraguan | Jawaban via website |
|----------|---------------------|
| Apakah berpengalaman? | Portfolio, proses kerja, trust bar |
| Layanan apa yang cocok? | 6 service cards + detail deliverable |
| Berapa biaya & durasi? | FAQ |
| Profesional atau sekadar kecil? | Hero, trust bar, studi kasus |
| Beda apa dengan PDF? | FAQ + satu link interaktif |

## 3. Website Structure

13 section untuk demo page penuh (`/demo/company-profile`):

| # | Section | Tujuan singkat |
|---|---------|----------------|
| 1 | Navbar | Navigasi + CTA selalu terlihat |
| 2 | Hero | Positioning 5 detik + CTA |
| 3 | Trust bar | Validasi cepat tanpa klaim berlebihan |
| 4 | Problem / positioning | Empati — "ini masalah saya" |
| 5 | Layanan utama (overview) | Scan 6 layanan |
| 6 | Detail layanan | Depth per layanan |
| 7 | Proses kerja | Kurangi ketakutan proses tidak jelas |
| 8 | Portfolio proyek | Bukti kerja dengan filter |
| 9 | Studi kasus singkat | Depth naratif |
| 10 | Testimoni skenario | Social proof tone — berlabel |
| 11 | FAQ | Jawab keberatan sebelum chat |
| 12 | Form inquiry / WhatsApp | Konversi utama |
| 13 | Footer | Trust closing + AppVibe backlink |

### Ringkasan per section (tujuan + CTA)

**Navbar** — Menu: Beranda, Layanan, Portfolio, Proses, FAQ, Kontak. CTA: Konsultasi.

**Hero** — Headline positioning + 2 CTA (Konsultasi Awal / Lihat Portfolio). Trust chips: pengalaman, fokus UMKM & B2B, pendekatan terukur.

**Trust bar** — 4 stat aman: 6 layanan inti · 68+ proyek contoh (simulasi) · 4 tahap proses · Form + WhatsApp.

**Problem** — 3 pain cards: layanan dijelaskan berulang, portfolio tidak tertata, tidak ada link resmi.

**Layanan overview** — Grid 3×2 service cards dengan ikon, judul, deskripsi 2 baris.

**Detail layanan** — Tab/accordion 6 layanan: deskripsi, cocok untuk siapa, deliverable, pain point.

**Proses kerja** — 4 langkah: Konsultasi awal → Diagnosis → Eksekusi → Serah terima.

**Portfolio** — Filter chips + grid cards. Klik → modal detail.

**Studi kasus** — 3 kartu: Challenge → Approach → Outcome naratif aman.

**Testimoni** — Label wajib: **Contoh testimoni skenario — bukan klien nyata**. 4 kartu persona dummy.

**FAQ** — Accordion 8 item.

**Form inquiry** — 6 field + WhatsApp card. Success state setelah submit.

**Footer** — Dark navy, legal note simulasi, AppVibe backlink.

## 6. UI/UX Blueprint

### Visual Style
- **Kesan:** Premium-light, profesional, terstruktur
- **Mood:** Tenang, meyakinkan, modern bisnis
- **Hindari:** Gradient berlebihan, stock photo wajah asli, animasi berlebihan, template UMKM murahan

### Palet Warna

| Token | Hex | Penggunaan |
|-------|-----|------------|
| Primary | `#0F172A` | Heading, footer bg |
| Secondary | `#2563EB` | Link, filter aktif |
| Accent | `#7C3AED` | Highlight, gradient hero |
| Accent alt | `#06B6D4` | Eyebrow, badge |
| Background | `#FFFFFF` | Main bg |
| Surface | `#F8FAFC` | Alternating section |
| Text | `#111827` | Body |
| Muted | `#64748B` | Deskripsi |
| Border | `#E2E8F0` | Card, input |
| WhatsApp | `#16A34A` | WA CTA |

**Gradient hero:** `linear-gradient(135deg, #0F172A 0%, #1E3A8A 45%, #4C1D95 100%)`

### Typography
- **Font:** Plus Jakarta Sans (selaras design system AppVibe)
- **H1:** 2.5–3.5rem desktop, bold
- **H2:** 1.75–2.25rem, semibold
- **Body:** 1rem, line-height 1.6–1.75
- **Eyebrow:** 0.75rem, uppercase, tracking-wide

### Layout Desktop
- Max width: 1280px (80rem)
- Grid: 12-column, gutter 24px
- Section padding: 80–120px vertical
- Hero: 55/45 split teks/visual
- Portfolio: 3 kolom + filter horizontal
- Form: 60/40 form + WA card

### Layout Mobile
- Single column, hamburger navbar
- Sticky CTA bottom setelah scroll 40%
- Filter horizontal scroll
- Form full-width, WA card di bawah

### Komponen Utama

| Komponen | Data | Interaksi |
|----------|------|-----------|
| Navbar | menu, logo | Sticky blur |
| Hero badge | 1 string | Static |
| Trust stat card | label + value | Static |
| Service card | title, desc, icon | Hover → detail |
| Process step | number, title, desc | Scroll reveal |
| Portfolio card | title, category, year | Click → modal |
| Case study card | challenge, outcome | Expand/link |
| Testimonial card | persona, quote, label | Static |
| FAQ accordion | Q&A | Expand/collapse |
| Inquiry form | 6 fields | Validate, success |
| WhatsApp card | prefilled msg | Open WA tab |
| Footer | links, legal | Static |

### Card Style
- Radius: 12–16px
- Shadow: subtle `0 1px 3px rgba(15,23,42,0.06)`
- Border: 1px `#E2E8F0`
- Hover: translateY(-2px) + shadow medium
- Image placeholder: gradient slate, no stock photo

### CTA Style
- **Primary:** Gradient `#2563EB → #7C3AED`, white text
- **Secondary:** Outline
- **Sticky mobile:** Konsultasi + WA icon
- **Placement:** Hero, post-layanan, post-portfolio, form, footer

## 9. Conversion Strategy

### CTA
- **Utama:** Jadwalkan Konsultasi Awal (form)
- **Sekunder:** Chat via WhatsApp

### Urutan persuasi
Empati (problem) → Solusi (layanan) → Bukti (portfolio + studi kasus) → Kejelasan (proses + FAQ) → Aksi (form + WA)

### Field form untuk inquiry berkualitas
Nama, bisnis, jenis kebutuhan (select), estimasi budget (select), pesan opsional.

**Select kebutuhan:** Audit Operasional, SOP, Sales Process, Digitalisasi, Pendampingan, Dashboard, Belum yakin.

**Select budget:** <10jt, 10–25jt, 25–50jt, >50jt, Belum tahu.

### WhatsApp prefilled message

```
Halo Arunika Konsultan, saya tertarik konsultasi operasional bisnis.

Nama: [nama]
Bisnis: [nama bisnis]
Kebutuhan: [jenis kebutuhan]
Budget: [kisaran]

Bisa dijadwalkan konsultasi awal?
```

### Risiko overclaim & pencegahan

| Risiko | Pencegahan |
|--------|------------|
| "500+ klien" | "68+ proyek contoh (simulasi)" |
| Testimoni palsu | Label "Contoh skenario — bukan klien nyata" |
| "Omzet naik X%" | Outcome naratif: "alur lebih rapi" |
| Foto klien asli | Avatar inisial CSS |
